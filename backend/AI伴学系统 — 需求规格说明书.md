# AI伴学系统 — 需求规格说明书

本说明书确立了“AI伴学与职业成长平台”实训项目的核心需求规范，旨在指导开发团队在四周内完成移动端（HarmonyOS NEXT）、管理后台（Vue 3）与后端服务（Spring Boot 3）的开发，并确保三端在功能与数据交互上保持一致。

------



## 1. 项目概述

本平台是一个面向高校学生的智能学习辅助系统。通过“移动端应用”为学生提供技能追踪、AI答疑和模拟面试服务；通过“Web管理端”为管理员提供用户和技能树的日常运营管理；通过“单体后端服务”连接数据库并桥接外部 AI 能力（Dify 与 DeepSeek API）。

## 3. 核心业务流程与功能说明

### 3.1 用户与认证

- **登录流**：客户端与管理端均通过 /api/auth/login 提交用户名和密码。后端验证通过后，生成具有 24 小时有效期的 JWT 令牌（Token）。
- **鉴权流**：除登录、注册接口外，所有请求需在请求头携带 Authorization: Bearer {Token}。后端通过拦截器进行 Token 解析和权限校验（ADMIN/STUDENT）。

### 3.2 技能点亮交互

- **点亮流**：移动端点击某个子技能的“点亮”按钮，向后端发送请求。
- **状态流转**：未开始（0） 学习中（1） 已掌握（2）。
- **一致性要求**：当移动端将状态修改为“已掌握”后，管理后台的仪表盘和用户技能分布图表需实时同步该统计数据的变更。

### 3.3 AI 智能辅导

- **交互机制**：移动端向后端发送用户提问。后端作为中转网关，调用 Dify 平台发布的 Chatflow API，获取 AI 的回复并返回给移动端。

### 3.4 模拟面试

- **面试前**：移动端传入技能ID，后端调用 AI 服务或预设题库，随机生成 3 道面试题。
- **面试中**：移动端逐题提交答案，后端将答案存入临时表，不即时打分，以保证流畅度。
- **面试后**：最后一题提交后，后端触发评测逻辑（通过 Dify 工作流或直接调用大模型），计算得出综合得分及改进建议，并将结果存入数据库，移动端跳转至报告页展现。

## 4. 简明数据库设计

系统采用单体关系型数据库（MySQL 8.0）存储业务数据，核心实体表结构设计如下：

### 表 1：sys_user（用户表）

| 字段名   | 类型         | 约束              | 说明                  |
| -------- | ------------ | ----------------- | --------------------- |
| id       | BIGINT       | PK, 自增          | 用户唯一标识          |
| username | VARCHAR(50)  | UNIQUE, NOT NULL  | 登录账号              |
| password | VARCHAR(255) | NOT NULL          | BCrypt 加密密文       |
| nickname | VARCHAR(50)  | -                 | 用户昵称              |
| role     | VARCHAR(20)  | DEFAULT 'STUDENT' | 角色：ADMIN / STUDENT |
| status   | TINYINT      | DEFAULT 1         | 状态：0-禁用，1-启用  |

### 表 2：skill_tree（技能树基本信息表）

| 字段名      | 类型         | 约束      | 说明                                                       |
| ----------- | ------------ | --------- | ---------------------------------------------------------- |
| id          | BIGINT       | PK, 自增  | 技能唯一标识                                               |
| name        | VARCHAR(100) | NOT NULL  | 技能节点名称                                               |
| description | TEXT         | -         | 技能描述                                                   |
| parent_id   | BIGINT       | DEFAULT 0 | 父节点 ID（0 表示根节点）                                  |
| category    | VARCHAR(50)  | NOT NULL  | 分类：FRONTEND / BACKEND / DATABASE / DEVOPS / SOFT_SKILLS |

### 表 3：user_skill（用户技能关联表）

| 字段名     | 类型     | 约束      | 说明                               |
| ---------- | -------- | --------- | ---------------------------------- |
| id         | BIGINT   | PK, 自增  | 关联唯一标识                       |
| user_id    | BIGINT   | NOT NULL  | 关联用户 ID                        |
| skill_id   | BIGINT   | NOT NULL  | 关联技能 ID                        |
| status     | TINYINT  | DEFAULT 0 | 状态：0-未开始，1-学习中，2-已掌握 |
| learned_at | DATETIME | -         | 掌握时间                           |

### 表 4：interview_session（面试会话表）

| 字段名       | 类型         | 约束                  | 说明                          |
| ------------ | ------------ | --------------------- | ----------------------------- |
| id           | BIGINT       | PK, 自增              | 会话唯一标识                  |
| user_id      | BIGINT       | NOT NULL              | 面试用户 ID                   |
| job_position | VARCHAR(100) | NOT NULL              | 目标岗位/关联技能             |
| score        | INT          | -                     | 面试总分（0-100）             |
| feedback     | TEXT         | -                     | AI 综合考评语                 |
| status       | VARCHAR(20)  | DEFAULT 'IN_PROGRESS' | 状态：IN_PROGRESS / COMPLETED |

### 表 5：interview_question（面试题目关联表）

| 字段名        | 类型   | 约束     | 说明         |
| ------------- | ------ | -------- | ------------ |
| id            | BIGINT | PK, 自增 | 题目唯一标识 |
| session_id    | BIGINT | NOT NULL | 关联会话 ID  |
| question_text | TEXT   | NOT NULL | 题目内容     |
| user_answer   | TEXT   | -        | 用户作答文本 |
| score         | INT    | -        | 单题得分     |

## 5. 三端数据交互契约 (API 规范)

### 5.1 统一响应格式

所有 HTTP 接口的返回数据需统一包裹在如下 JSON 实体中：

```
{
  "code": 200, 
  "message": "success",
  "data": {}
}
```

*常见错误码*：400（参数错误）、401（未登录/Token过期）、403（权限不足）、500（服务器异常）。

### 5.2 核心 API 清单

| 请求方法 | 路由地址                | 请求体/入参          | 说明                                    |
| -------- | ----------------------- | -------------------- | --------------------------------------- |
| **POST** | /api/auth/login         | {username, password} | 用户登录，返回用户信息及 Token          |
| **GET**  | /api/skills/tree        | -                    | 获取完整树状结构的技能列表              |
| **POST** | /api/skills/toggle/{id} | -                    | 切换点亮状态，改变 user_skill 表数据    |
| **POST** | /api/ai/chat/message    | {message}            | 提交对话消息，返回 AI 回复文本          |
| **POST** | /api/interview/start    | {jobPosition}        | 开启面试，返回 SessionID 及首题内容     |
| **POST** | /api/interview/answer   | {sessionId, answer}  | 提交当前题目回答，返回下一题或结束标识  |
| **GET**  | /api/dashboard          | -                    | 获取管理后台统计汇总（仅限 ADMIN 访问） |

## 6. 开发环境与集成参数

- **本地联调网络地址配置**：
  - **Vue 管理后台**：API 代理指向 http://localhost:8080（网关或单体后端服务端口）。
  - **HarmonyOS 模拟器**：网络连接请求指向宿主机地址 http://10.0.2.2:8080（不可直连 localhost）。
- **数据库初始化要求**：
  - 开发前需统一导入初始账号（如 admin/admin123 作为管理员，student01/student123 作为学生）及 5 大顶级分类和 15 个基础子技能节点，以确保三端展现的数据底座完全一致。
